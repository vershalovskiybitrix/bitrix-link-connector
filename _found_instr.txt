## Экран 1 — категория и разделы

**"1 версия приложения"** — оставить **«Использовать REST API»** (уже выбрано по умолчанию). Остальные варианты — для других типов решений (CRM-конфигураторы, BI-отчёты, шаблоны сайтов и т.д.), это не про нас: приложение целиком построено на `BX24.callMethod`.

**«Приложение будет работать с разделами системы»** → выбрать:
- **Открытые линии** — основная функциональность (коннекторы)
- **Контакт-центр** — плитка приложения там
- **Встраивание приложений** — используется `placement.bind`/`placement.info`

Это те разделы, с которыми приложение реально взаимодействует (соответствует фактическим вызовам `imopenlines.*`, `imconnector.*`, `placement.*`, `contact_center`).

**«Минимально необходимый тариф»** — нужно свериться вживую по [таблице тарифов](https://www.bitrix24.ru/prices/compare_cloud_plans.php): открытые линии и коннекторы доступны не на всех планах (обычно не на бесплатном). Какой именно план — я не могу сказать точно, тарифная сетка меняется; посмотри там, на каком плане впервые появляются «Открытые линии»/«Контакт-центр», и укажи его как минимальный.

## Экран 2 — встраивание и доставка файла

- **«Добавлять свою страницу и пункт в главном меню»** — оставить ☑ **включённым**. Это даёт пользователю постоянный способ открыть приложение повторно (важно для статичных приложений — у них нет install-хука, и без этого пункта меню после первой установки человеку будет неоткуда повторно зайти на экран «Активация»). Не забудь указать название пункта меню (например, «Свои ссылки на мессенджеры») в описании приложения — система явно требует это сделать.
- **«Встраиваться в BitrixMobile»** — оставить ☐ выключенным (мобильных доработок нет).
- **«Встраивать виджеты в интерфейс»** — оставить ☐ выключенным (это другой механизм встраивания, не используется).
- **«Может устанавливать любой пользователь»** — оставить ☐ выключенным (по умолчанию ставит только админ — логично, т.к. `imconnector.register` и подобные методы требуют админских прав).

**Ссылки vs Загрузка** — нужен только один из двух блоков («или»). Поля «Ссылка на приложение / установочное / настройки» — это для серверных приложений с собственным хостингом, у нас их оставляем пустыми (`https://`). Используем **«Загрузить приложение»** — там уже подгружен актуальный `app.zip` (8 КБ), всё верно.
